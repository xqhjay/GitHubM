# AI 助手开发方向规划

> 基于当前 v212 版本功能分析，列出下一阶段优先级高、价值明确的开发方向。

---

## 🔴 高优先级（用户痛点 / 高频使用）

### 1. 代码 Diff 对比视图
- **背景**：AI 建议修改代码时，用户当前只能看到 AI 生成的新版代码，无法直观对比修改前后差异。
- **方案**：
  - 对 AI 返回的 diff patch 进行解析，以行级 diff 形式展示
  - 支持高亮增、删、修改的代码行
  - 每处 diff 提供「接受」/「拒绝」/「修改」三种操作
  - 接受后可一键提交到当前分支
- **价值**：大幅降低 AI 代码的审阅成本，提升代码安全性

### 2. Agent 自主执行模式
- **背景**：当前 AI 执行多步计划时，每一步都需要用户点击确认，操作繁琐。
- **方案**：
  - 新增「自主执行」开关，AI 可自动执行安全级别较低的工具调用
  - 危险操作（如 push、merge、删除）仍需用户确认
  - 实时显示执行进度，支持随时暂停
  - 执行完成后给出汇总报告
- **价值**：将 AI 从“协助工具”提升为“自动化助手”，效率提升 3~5 倍

### 3. 会话搜索与标签管理
- **背景**：历史对话面板已支持批量删除，但缺乏搜索和分类能力。
- **方案**：
  - 历史面板顶部添加搜索框，支持按标题/内容搜索
  - 支持给会话添加自定义标签（如 bugfix、feature、refactor）
  - 支持按标签/时间/仓库筛选
  - 最近使用的标签放在快捷入口
- **价值**：历史对话数量多时，快速定位所需信息

### 4. 知识库 / RAG 上下文
- **背景**：AI 对项目的理解只能通过当前文件浏览器拖入的几个文件，缺乏项目级别的上下文。
- **方案**：
  - 支持上传项目文档（README、API 文档、设计文档等）作为知识库
  - 对知识库文档进行向量化，实现 RAG 检索
  - AI 回答时自动引用知识库中相关内容
  - 支持指定哪些文件加入/排除知识库
- **价值**：AI 对项目的理解从“拖入几个文件”提升为“了解整个项目”

## 🟡 中优先级（体验优化 / 效率提升）

### 5. 快捷键系统
- **方案**：
  - `Cmd/Ctrl + K` 快速打开会话搜索
  - `Cmd/Ctrl + N` 新建会话
  - `Esc` 关闭弹窗/面板
  - `Shift + Enter` 换行 / `Enter` 发送（可切换）
  - `/` 快速激活命令菜单（类似 VS Code Command Palette）
- **价值**：熟练用户操作效率大幅提升

### 6. 会话导出与分享
- **方案**：
  - 支持将完整会话导出为 Markdown / PDF 文件
  - 生成可分享的临时链接（无需登录即可查看）
  - 导出时保留代码高亮、任务进度等格式
- **价值**：方便将 AI 分析结果分享给团队或留存档案

### 7. 代码审查助手
- **方案**：
  - 在 PR 列表页面新增「AI 审查」按钮
  - AI 自动分析 PR diff，检查潜在问题
  - 生成审查报告（安全、性能、可读性、最佳实践）
  - 支持在 AI 助手中继续讨论审查发现的问题
- **价值**：降低代码 review 成本，提升代码质量

### 8. 移动端输入体验优化
- **背景**：当前移动端输入框在键盘弹出时存在视口被挤压的问题。
- **方案**：
  - 软键盘弹出时自动调整消息区视口（利用 Visual Viewport API）
  - 输入框增加「全屏扩展」模式，便于长文本编辑
  - 支持语音输入（移动端先行）
  - 快捷提示词在移动端以底部横向滚动条形式展示
- **价值**：移动端是本应用主场景，输入体验直接决定用户留存

## 🟢 低优先级（增值功能 / 未来尝试）

### 9. 使用量统计与限额管理
- **方案**：
  - 个人中心新增 Token 消耗统计面板
  - 按模型、按日期统计调用次数与 Token 数量
  - 支持设置日/月 API 使用上限，接近时提醒
  - 提供使用报告导出
- **价值**：帮助用户掌控 API 成本

### 10. 自定义工具市场
- **方案**：
  - 提供预设工具模板（如自动化测试、生成提交日志、国际化检查）
  - 支持用户定义自己的工具（JSON Schema 描述）
  - 工具支持引入外部 API 调用
- **价值**：极大扩展 AI 助手的能力边界

### 11. 深色/浅色主题切换
- **方案**：
  - Android 端状态栏/导航栏主题已修复为默认浅色
  - 前端增加完整的主题切换功能（跟随系统 / 浅色 / 深色）
  - 主题状态持久化到 localStorage
  - 所有组件主题变量使用 CSS 变量统一管理
- **价值**：尊重用户使用偏好

- 代码审查助手 → 自定义工具市场 → 使用量统计 → 会话导出分享
